## 1.1.1 Conceptos básicos

**Una base de datos (BD)** es un conjunto organizado de información estructurada que permite almacenar, gestionar y recuperar datos de forma eficiente.
→ *La agenda de contactos del móvil*

Antes de la aparición de las bases de datos modernas, las organizaciones utilizaban **ficheros planos**.

**Los ficheros planos:** (archivos como `.txt` o `.csv`) almacenan datos en formato simple, sin estructura relacional ni control avanzado. Aunque son fáciles de usar, presentan varios **problemas importantes** cuando se manejan grandes volúmenes de información:

- Inconsistencia de datos
- Dificultad de acceso
- Falta de integridad
- Problemas de seguridad
- Problemas de concurrencia
- Dependencia del programa

Con el crecimiento de la informática y la digitalización de las empresas, **surgió la necesidad de sistemas especializados capaces de organizar y proteger la información**.

---

## 1.1.2 Sistema de Gestión de Bases de Datos (SGBD)

El **Sistema de Gestión de Bases de Datos (SGBD)** es un **software que permite crear, administrar, organizar y manipular bases de datos** de manera eficiente, segura y estructurada. 
→ *Básicamente, es el programa que actúa como intermediario entre los usuarios o aplicaciones y los datos almacenados.*

Algunas funciones principales de un SGBD son:

- Almacenar información.
- Permitir consultas y modificaciones.
- Controlar el acceso de usuarios.
- Garantizar la integridad de los datos.
- Gestionar copias de seguridad y recuperación.

**Ejemplos de SGBD:**

| SGBD | Tipo |
| --- | --- |
| MySQL | Relacional |
| PostgreSQL | Relacional |
| Oracle | Relacional |
| Microsoft SQL Server | Relacional |
| MongoDB | No relacional |
| Cassandra | No relacional |

**Características:**

- **Integridad:** los datos siempre tienen sentido
→ *En Amazon no puedes comprar sin registrarte, de forma que ningún pedido queda “huérfano”.*
- **Seguridad:** controla accesos
→ *El administrativo de un hospital no puede ver diagnósticos médicos*
- **Escalabilidad:** soporta muchísimos usuarios
→ *Netflix puede reproducir millones de películas a la vez sin caerse*
- **Acceso concurrente:** varios usuarios acceden en paralelo
→ *Gmail permite enviar correos simultáneamente a millones de usuarios*

---

## 1.1.3 SGBD VS Ficheros planos

**Ficheros planos**

- Integridad manual, propensa a errores
- Seguridad básica o nula
- Búsquedas limitadas o ineficientes
- Riesgo de conflictos en concurrencia
- Difícil escalabilidad

**SGBD**

- Reglas y restricciones integrados
- Autentificación, permisos, controles de acceso
- Lenguajes declarativos (SQL) optimizadores
- Control de acceso concurrente
- Escalable en datos/usuarios

## 1.1.4 Evolución histórica de las Bases de Datos

### Años 70-80

Modelo Relacional (SQL): tablas, lenguaje SQL.
*Se popularizó el modelo relacional de Edgar F. Codd, que organizaba los datos en tablas con filas y columnas, simplificando enormemente la consulta mediante SQL. Por ejemplo, bancos con IBM para cuentas corrientes.*

### Años 90-2000

Orientadas a objetos y distribuidas: soportan datos complejos.
*Aparecieron sistemas orientados a objetos y posteriormente las bases de datos distribuidas, adaptándose a las nuevas necesidades empresariales. POr ejemplo, videojuegos online con inventarios y personajes.*

### 2010+

NoSQL, nube, BigData: flexibles y rápidas

*El auge de internet y el big data impulsó las bases de datos NoSQL, diseñadas para manejar grandes volúmenes de información no estructurada. Por ejemplo, Facebook y X procesando millones de publicaciones por segundo.*
