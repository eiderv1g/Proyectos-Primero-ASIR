# 03 - Intro a AWS

# 1. INTRODUCCIÓN AWS

![image.png](03%20-%20Intro%20a%20AWS/image.png)

**AWS (Amazon Web Services)** es una plataforma de **computación en la nube** que ofrece servicios como almacenamiento, servidores, bases de datos, inteligencia artificial y más, a través de internet bajo un modelo de pago por uso. Sus principales características son:

- **Escalabilidad**: puedes aumentar o reducir recursos según la demanda.
- **Pago por uso**: solo pagas por lo que utilizas, sin inversión inicial en hardware.
- **Alta disponibilidad**: infraestructura distribuida globalmente con centros de datos en múltiples regiones.
- **Seguridad avanzada**: incluye cifrado, control de acceso y cumplimiento de normativas internacionales.
- **Amplia variedad de servicios**: desde máquinas virtuales (EC2) hasta almacenamiento (S3), IA, IoT, etc.
- **Elasticidad**: adaptación automática a cargas de trabajo variables.
- **Accesibilidad global**: se puede acceder desde cualquier lugar con conexión a internet

---

# 2. ORÍGENES E HISTORIA

Entre los años **2000 y 2002**, Amazon necesitó mejorar su propia plataforma, así que empezó a desarrollar una nueva infraestructura interna. En el **2006**, lanzó oficialmente AWS con servicios clave como S3 (almacenamiento) y EC2 (computación). Del **2010** en adelante, AWS creció rápidamente y pasó a convertirse en líder del mercado de cloud computing. 

A día de hoy, AWS es una de las plataformas más usadas por empresas, startups y gobiernos en todo el mundo. A pesar de la competencia, AWS sigue siendo líder mundial.

# 4. ALGUNAS PREGUNTAS CLAVE

### ¿Qué es una región AWS?

Es una ubicación geográfica física en el mundo donde AWS tiene centros de datos. Cada región funciona de manera independiente y contiene múltiples centros de datos.

### ¿Qué es una Availability Zone (AZ)?

Una AZ (Zona de Disponibilidad) es un centro de datos (o conjunto) dentro de una región:

- Cada región tiene varias AZs
- Están aisladas entre sí (así un fallo no afecta a todas)
- Están conectadas con baja latencia

**Por lo tanto, región = zona del mundo | AZ = centro de datos dentro de esa zona**

---

### ¿Qué criterios usar para elegir una región?

- **Latencia:** cuanto más cerca esté el usuario, mejor rendimiento
- **Coste:** los precios varían según la región
- Regulación y datos: algunas leyes obligan a guardar datos en ciertos países (*ej: GDPR en Europa*)
- **Servicios disponibles:** no todas las regiones cuentan con todos los servicios
- **Alta disponibilidad:** elegir regiones con múltiples AZs para mejorar resiliencia
- **Ubicación de usuarios:** escoger la región más cercana al público objetivo

# 5. EJEMPLO DE ARQUITECTURA BÁSICA EN AWS

![image.png](03%20-%20Intro%20a%20AWS/image%202.png)

### Usuarios

Son las personas o aplicaciones que acceden al sistema (web, app móvil, etc.).

### VPC (Virtual Private Cloud)

Es la **red privada** dentro de AWS. Aísla y protege los recursos Y Controla tráfico, IPs y seguridad.

### Amazon EC2 (Informática)

Servidor virtual donde corre la aplicación que procesa las peticiones de los usuarios. Viene a ser el “cerebro” de la aplicación.

### Amazon DynamoDB (Base de datos)

Base de datos NoSQL rápida y escalable. Guarda información estructurada (usuarios, datos, etc.).

### Amazon S3 (Almacenamiento)

Guarda archivos (imágenes, vídeos, documentos). Ofrece alta durabilidad y disponibilidad.

## 🔄 ¿Cómo funciona el flujo?

1. 👤 Usuario envía una solicitud
2. 🌐 Entra a la VPC
3. 💻 EC2 procesa la solicitud
4. 🗄️ Consulta o guarda datos en DynamoDB
5. 📦 Almacena o recupera archivos desde S3
6. 📤 Devuelve la respuesta al usuario
